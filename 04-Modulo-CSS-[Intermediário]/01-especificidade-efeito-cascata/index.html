Efeito Cascata:

1.0 - Exemplo [HTML]

Código padrão no HTML para exemplificar o efeito cascata natual.

1.1 - Exemplo [CSS]

Pela ordem de preferência da cascata, as configurações de estilo da tag link:css se aplicam de cima para baixo.
OBS: Classes foram usadas no exemplo.

2.0 - Exemplo

Estilos internos <style> </style> são priorizados no final da visualização do usuário por causa da leitura do código.
Primeiro o VSCode lê o link:css e todos os códigos dessa página em sequência de cima para baixo, depois lê todos os estilos internos da página html. Já que o link:css está sempre acima do <style> </style> o que prevalece ao final para o usuário são as configurações de estilo do <style> </style>.

3.0 Exemplo

Por sua vez, estilos inline estão ainda mais abaixo do que os estilos internos, mostrando assim para o usuário apenas as configurações de estilo inline

Especificidade:

  • ID's = Primeiro nível de especificidade, recebe prioridade nas configurações de estilo
  • Classes, Atributos e Pseudo-Classes = Segundo nível de especificidade, recebe menos prioridade que os ID's.
  • Elementos e Pseudo-Elementos = Terceiro nível de especificidade, recebe o nível mais baixo de prioridade.
  • !important = Prioridade Absoluta. Tem peso máximo de prioridade dentre todas as outras, mas tem o mesmo peso de um outro !important [usar com cuidado!].
