Tratamento de Erros

try{}

● Serve para verificar se houve algum erro. Os códigos ficam dentro das chaves do try.

Ex:

            
                try{
                    const lorem1 = function(){
                        alert('Olá, Mundo!')
                    }
                    lorem1()
                }

ATENÇÃO: Mesmo que uma função(com erros) seja escrita fora do try{}, ele ainda mostra o erro dessa função, desde que a chamada esteja dentro do try.
Ex:

            
                const loremFunction(){
                    alert('Seja bem vindo, Quester Guga!')
                }
                try{
                    loremFunction()
                }

catch(error){}

● Dentro das chaves, os erros do try{} serão tratados.

Ex:

            
                try{
                    const loremFunction(){
                        alert('Seja bem vindo, Quester Guga!')
                    }
                    loremFunction()
                } catch(error){
                    console.log(error)
        
                }

finally{}

● Serve para executar uma ação final independente da execução do try/catch. O finally{} é muito indicado para fechar uma conexão de banco de dados ou fechar arquivos que foram abertos para leitura, caso contrário, se houver algum erro no try ou no catch, a conexão com o banco ou o arquivo ficarão indefinidamente abertos, podendo gerar erros graves no sistema.

Ex:

            
                try{
                    const loremFunction(){
                        alert('Seja bem vindo, Quester Guga!')
                    }
                    loremFunction()
        
                } catch(error){
                    alert('Erro de funcionamento. O erro rolou aqui:')
                    console.log(error)
        
                } finally{
                    alert('Vamos consertar tudo, ficará tudo bem muito em breve.
                    Curta nosso joguinho enquanto isso:')
        
                    jogoDaCobrinha()
                }

throw

● Tradução: "Arremeçar"/"jogar" um erro pro catch. "Cria exceções".

Exemplo no código JS completo.
Manter tudo como está o script roda normalmente.
Alterar os valores booleanos true para false vai estourar o erro 001.
Alterar os valores das variáveis de Boolean para numeric ou qualquer outro vai estourar o erro 002 -> twrow.
