Stateless vs Statefull

Stateless (Sem Estado):

  • São Componentes que trabalham só com propriedades (props) e não trabalham com estado.
  • São considerados Componentes sem lógica complexa.
  • Servem muito mais pra renderizar componentes na tela e terão sua "lógica" toda em torno de suas próprias props.
  • São bem fáceis de ler, entender e testar (ao menos deveriam ser).

Statefull (Com Estado):

  • São Componentes que podem ter tando propriedades/props quanto estado/state.
  • Geralmente são responsáveis pela comunicação entre cliente-servidor, processando dados e respondendo aos eventos dos usuários (ex: consumindo API e trazendo uma lista de usuários pra mostrar na tela).
  • Esse tipo de Lógica deve ser encapsulada em um número pequeno de Componentes Statefull
    enquanto que toda a Lógica de formatação e visualização deve ser transferida pro máximo de Componentes Stateless possível.
